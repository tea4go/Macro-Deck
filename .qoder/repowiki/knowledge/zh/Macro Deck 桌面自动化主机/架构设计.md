- **混合宿主架构**：在 `Program.cs` 中初始化 Windows Forms 消息循环，并通过 `ServerStartup` 嵌入 ASP.NET Core 管道，实现本地 UI 与远程 Web/移动端（WebSocket/REST）的统一接入。
- **集中式依赖治理**：通过根目录的 `Directory.Packages.props` 启用中央包管理（CPM），强制所有子模块（主程序、测试）使用统一的 NuGet 依赖版本，确保生态一致性。
- **跨进程单实例协调**：利用命名管道（`MacroDeckPipeServer/Client`）在 `Program` 启动阶段进行互斥检查，实现多实例间的通信与窗口激活，保障系统资源独占性。
- **模块化插件边界**：通过 `InternalPlugins` 目录将核心功能（如动作按钮、变量、文件夹）以插件形式解耦，由 `PluginManager` 统一加载，形成“内核+扩展”的动态架构。
