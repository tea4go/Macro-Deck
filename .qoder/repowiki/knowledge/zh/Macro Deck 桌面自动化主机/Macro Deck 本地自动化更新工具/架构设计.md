- **单一入口架构**：以 `update-macrodeck-local.ps1` 为核心，通过参数（`-Install`, `-Build`, `-Version`, `-Status`）驱动不同的运维子流程。
- **外部依赖集成**：深度依赖 `gh` (GitHub CLI) 进行 API 交互，包括查询工作流状态 (`gh api`)、触发构建 (`gh workflow run`) 及下载构件 (`gh run download`)。
- **原子化更新策略**：采用“停止进程 -> 覆盖核心文件（跳过用户数据目录） -> 重启进程”的流程，确保本地环境在更新过程中的安全性与一致性。
- **状态轮询机制**：在触发新构建后，通过循环调用 API 轮询运行状态，并设置超时阈值以处理异步构建过程。
