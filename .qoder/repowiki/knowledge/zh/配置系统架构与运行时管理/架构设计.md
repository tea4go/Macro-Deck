- **入口与引导层**：`Program.cs` 作为应用入口，负责全局异常捕获、单实例检测（通过 `MacroDeckPipeClient/Server`）及 `ApplicationPaths` 的路径初始化。
- **核心控制层**：`MacroDeck.cs` 充当静态单例控制器，按严格顺序编排日志、语言、热键、变量、插件、图标、配置文件及网络服务的加载流程。
- **配置管理层**：采用 `MainConfiguration` 结合 `Newtonsoft.Json` 实现 JSON 持久化，并通过 `StartParameters`（基于 `CommandLineParser`）支持 CLI 参数覆盖；`ApplicationPaths` 统一管理“便携模式”与“安装模式”的双轨制存储路径。
- **集成边界**：通过 `ISerializableConfiguration` 规范插件配置，利用 `MacroDeckLogger` 集成 Serilog 进行集中式日志记录，并通过 `MainWindow` 承载 UI 线程同步上下文。
