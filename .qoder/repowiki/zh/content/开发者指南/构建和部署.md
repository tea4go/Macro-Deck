## 详细组件分析

### Windows 构建自动化系统
- run_win.ps1 PowerShell 脚本
  - **三种运行模式**：支持 -Build（构建和发布）、-Run（运行已发布的版本）、-Portable（便携模式运行）
  - **智能参数处理**：完整的帮助信息，支持 -Configuration（默认 Release）和 -Output（默认 C:\Program Files\Macro Deck）
  - **管理员权限检测**：自动检测是否以管理员身份运行，对受保护目录写入进行安全检查
  - **进程管理**：构建前自动退出正在运行的旧版本，避免文件被锁
  - **依赖恢复**：自动执行 dotnet restore 确保所有 NuGet 包正确下载
  - **自包含发布**：使用 dotnet publish 生成 Release/win-x64 自包含部署
  - **便携启动脚本**：自动生成启动.bat批处理文件，支持便携模式启动
  - **错误处理**：包含 .NET SDK 检测和构建失败处理
- 传统构建脚本
  - build.ps1 脚本已被 run_win.ps1 完全替代，提供更完整的功能集

**更新** run_win.ps1 脚本提供了 Windows 环境下的完整构建自动化功能，包括三种运行模式、管理员权限检测、进程管理、智能参数处理、依赖恢复、自包含发布和便携启动脚本生成。

## 故障排除指南
- 安装失败或缺少运行时
  - 检查安装器是否正确触发 VC++ 运行时安装。
  - 确认防火墙规则已添加，必要时手动放行进程。
- 权限问题
  - app.manifest 默认 asInvoker，若需管理员权限请谨慎修改。
  - run_win.ps1 会自动检测管理员权限，避免在受保护目录写入失败。
- 日志与诊断
  - 启动参数可在 launchSettings.json 中配置，便于开启调试控制台与日志级别。
  - 关注 Serilog/Sentry 配置（工程引用存在，实际行为取决于运行环境）。
- PowerShell 脚本执行失败
  - 确认 .NET SDK 已正确安装且版本兼容。
  - 检查 PowerShell 执行策略设置。
  - 验证脚本权限和网络访问能力。
  - run_win.ps1 提供详细的错误处理和帮助信息。
- 端到端测试失败
  - 确认 Macro Deck 已正确安装在默认路径。
  - 检查 devices.json 中的客户端 ID 配置。
  - 验证防火墙未阻止 8191 端口。
- 构建中断问题
  - run_win.ps1 会在构建前自动停止旧版本进程，避免文件被锁。
  - 管理员权限检测确保对受保护目录的安全访问。

### run_win.ps1 脚本详细使用说明

#### 三种运行模式
- **-Build 模式**：执行完整的构建和发布流程
  - 自动检测 .NET SDK
  - 管理员权限检查
  - 依赖恢复
  - 自包含发布到指定输出目录
  - 生成启动.bat批处理文件
- **-Run 模式**：运行已发布的最新版本
  - 检查输出目录和可执行文件是否存在
  - 自动停止旧版本进程
  - 支持便携模式运行
- **-Portable 模式**：便携模式运行（与 -Run 配合使用）

#### 参数说明
- `-Build`：必需参数，执行构建和发布
- `-Run`：可选参数，运行已发布的版本
- `-Portable`：可选参数，便携模式运行
- `-Configuration`：构建配置，默认 Release
- `-Output`：输出目录，默认 C:\Program Files\Macro Deck

#### 安全特性
- 管理员权限检测：对受保护目录写入进行安全检查
- 进程管理：自动停止旧版本进程，避免文件被锁
- 错误处理：详细的错误信息和帮助提示
